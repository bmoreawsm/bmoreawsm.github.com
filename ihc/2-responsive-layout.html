Responsive Layout

Some visitors have more screen space than others
In this lesson, we'll build a layout that changes based on screen size

1. Simple Layout

Before we get to the responsive component, let's implement a simple layout. Just a header, sidebar, and footer.

First, let's setup the html, re-using our index.html page:

<body>
  <div class='wrapper'>
    <div class='sidebar'>
      <ul>
        <li><a href='#'>Link one</a></li>
        <li><a href='#'>Link two</a></li>
        <li><a href='#'>Link three</a></li>
      </ul>
    </div>
    <div class='header'>
      <h1>Site title</h1>
    </div>
    <div class='content'>
      <p>Enough lorem ipsum to make the page scroll.</p>
    </div>
    <div class='footer'>
      Made by Me!
    </div>
  </div>
</body>

And the css for the basic layout:

.wrapper {
  margin: auto;
}
.wrapper, .footer {
  width: 600px;
}
.header, .content {
  width: 450px;
}
.sidebar {
  width: 150px;
}
.footer {
  clear: both;
}
.header, .sidebar, .content {
  float: left;
}

2. Responsive CSS

We'd like to have a more fluid layout when someone visits on a smaller device. Since we have a 600px layout, our cutoff point will be 600px.

To do this, we're going to use a css feature called media queries. Here is our css for the media query:

@media (max-width: 600px) {
  .wrapper, .footer {
    width: 100%;
  }
  .header, .content {
    width: 75%;
  }
  .sidebar {
    width: 25%;
  }
}

The @media tag can respond to a number of different rules, like min-width, orientation, and even the device width.

3. Experiment

Experiment with different media queries and layouts. For example, could you bump up the size of the layout for people with very large screens? What about putting the sidebar at the bottom for a very small screen?

What about responsive typography? How would you change font size for different devices? What else could you change on a site based on size?

Check out the "column-count" and "column-gap" css3 properties. Try those on your site for people with large screens.
